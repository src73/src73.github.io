web programmer and user experience designer

Working independently on the user's experience, I have composed interfaces for several real-world applications. Included below are interactive proofs of concept to demonstrate a possible UX to my collaborators. While these concepts were implemented in the real world, the code used here is not production ready.


Objective: Case's Elementary Computer Programming class wants an online homework help intelligent tutoring system. To overcome student's hesitancies of asking for help, a user experience need to make students want to come, stay, and return.

This demo explores how bright colors, funny headlines, and simplicity can combine to create an interface that follows the mental scripts of a live human tutor. Hopefully, this interface, combined with useful information, will leave the user feeling that this tutoring system is a safe, helpful place. note: any username / password works.



Objective: After a year of development, the management console of Clipboard was impossible for even its developers to use. A user experience had to be created that allowed high school band directors to navigate the custom developed student management system and quickly accomplish their tasks on multiple devices.

This demo proposes an interface that has a common look, feel, capability, and usability across multiple devices. Large, similar buttons across all devices and screen widths attempt to invite touch, not be cumbersome for mouse, and be consistent. The placement of the navigation on the pages follows the scripts of other common software. Where the navigation may not be obvious, the consistent colors of the navigation prompt the user to click there for navigation.

This demo targets the common task of printing a students' arrival time at an event. As a result, only the Time / Select Event link works.
